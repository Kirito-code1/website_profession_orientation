# ПрофПуть — Платформа профориентации

Платформа для помощи абитуриентам в выборе профессии и поступления в вуз.

## Особенности

- **Тест на профориентацию** — 15 вопросов с персональными рекомендациями
- **Каталог факультетов и вузов** — база учебных заведений
- **Профиль пользователя** — сохранение результатов тестов и избранных профессий/факультетов
- **Аутентификация** — вход и регистрация через Supabase
- **Защита маршрутов** — только авторизованные пользователи могут проходить тест

## Запуск проекта

```bash
# Установка зависимостей
npm install

# Запуск dev-сервера
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000) в браузере.

## Настройка Supabase

### 1. Создайте проект
Зайдите на [supabase.com](https://supabase.com) и создайте новый проект.

### 2. Добавьте переменные окружения
В файле `.env.local` добавьте:
```env
NEXT_PUBLIC_SUPABASE_URL=ваш_url_из_settings
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш_anon_key_из_settings
```

### 3. Заполните базу данных
1. Откройте **SQL Editor** в Supabase Dashboard
2. Скопируйте содержимое `supabase_seed_data.sql`
3. Выполните скрипт

Это создаст:
- 15 вопросов для теста на профориентацию
- 75 вариантов ответов (по 5 на каждый вопрос)
- RLS политики для безопасного доступа к данным

### 4. Отключите подтверждение email
1. Перейдите в **Authentication → Providers → Email**
2. Отключите **"Confirm email"**
3. Сохраните изменения

Теперь пользователи могут регистрироваться без подтверждения по email.

## Основные функции

### Аутентификация
- Вход через email/пароль
- Регистрация с созданием записи в таблице `users`
- Сессии через Supabase Auth
- Защита маршрутов (тест доступен только авторизованным)

### Профиль пользователя
- **Сохранённое** — профессии и факультеты в избранном
- **Результаты тестов** — история прохождения с процентами
- **Настройки** — информация аккаунта и выход

### Тест на профориентацию
- 15 вопросов с 5 вариантами ответов
- Подсчёт процента выполнения
- Сохранение результатов в `career_test_results`
- Модалка для неавторизованных пользователей

### API Endpoints
| Method | Endpoint | Описание |
|--------|----------|----------|
| GET | `/api/test` | Получение всех вопросов теста |
| POST | `/api/test/results` | Сохранение результата теста |
| GET | `/api/test/results?user_id=...` | Получение результатов пользователя |
| GET | `/api/proffesions` | Получение всех профессий |
| GET | `/api/faculty` | Получение факультетов |
| GET | `/api/users/email/[email]` | Поиск пользователя по email |
| GET | `/api/users/id/[id]` | Получение пользователя по ID |
